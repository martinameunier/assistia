Installation and user documentation

Assistia installs and starts a local AI environment with Ollama and Open WebUI directly on your computer.

1. Understanding the tools used

Assistia brings together several tools so you can chat with artificial intelligence models from a local web interface.

Ollama

Ollama downloads and runs AI models on your machine. Assistia starts its local API on 127.0.0.1:11434 and prepares the qwen3:4b model.

Open WebUI

Open WebUI is the chat interface displayed in your browser. Assistia starts it on 127.0.0.1:8080 and connects it to Ollama.

Python 3.11 and uv

Open WebUI runs in a local Python environment managed by Assistia. If Python 3.11 is not available, Assistia installs uv, fetches Python 3.11, then installs Open WebUI in that environment.

In short: Ollama runs the models, Open WebUI provides the interface, and Assistia installs and starts the required components.

2. Install missing components

In Assistia, open Settings, then click Install missing components. The application first checks what is already present before launching any installation.

  • If Ollama is already installed, its installation is skipped.
  • If Open WebUI is already installed, its installation is skipped.
  • If a component is missing, Assistia installs only that component.

Ollama and Open WebUI remain third-party software subject to their own licenses and terms of use.

3. Settings in Assistia

The Settings button is located at the bottom right of the window. It lets you configure executable paths and install missing components.

Ollama path

Leave this field empty to use automatic detection. You can also enter the full path to the Ollama executable or to the folder that contains it.

Examples
  • WindowsC:\Users\you\AppData\Local\Programs\Ollama\ollama.exe
  • macOS/usr/local/bin/ollama
  • Linux/usr/bin/ollama

Open WebUI path

Leave this field empty to use Assistia's managed local installation. You can also provide a manually installed Open WebUI executable.

Examples
  • WindowsC:\Users\you\AppData\Roaming\Python\Scripts\open-webui.exe
  • macOS/usr/local/bin/open-webui
  • Linux/usr/local/bin/open-webui

4. Start and use Open WebUI

Start services

  1. Make sure missing components have been installed from the Settings menu.
  2. Click Start.
  3. Assistia starts Ollama, prepares the qwen3:4b model, then launches Open WebUI.
  4. When Open WebUI is available, click Open WebUI.

If Open WebUI displays an error on first launch, wait a few minutes and refresh the page. Model download or service initialization can take some time.

Add a new model

  1. In Open WebUI, open the model selector or the administration settings depending on your user profile.
  2. Search for an Ollama-compatible model, for example llama3.2 or mistral. See the model table
  3. Start downloading the model.
  4. Wait for the installation to finish before starting a conversation.

Customize a model with a system prompt

  1. In Open WebUI, go to Workspace.
  2. Click + New Model.
  3. Enter a model name, for example: My custom AI assistant.
  4. Select an available base model.
  5. Add a system prompt to customize your model.
  6. Save the configuration, then select this custom model in a new conversation.
