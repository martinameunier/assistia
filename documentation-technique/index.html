Architecture générale

Interface React

src/app.tsx orchestre la navigation, les pages, les hooks de statut, les installations et les providers de conversation.

Pont Tauri

src/services/tauris.ts centralise les appels invoke vers les commandes Rust exposées par src-tauri/src/main.rs.

Runtime local

Le backend démarre et supervise Ollama sur 127.0.0.1:11434 et ComfyUI sur 127.0.0.1:8188.

Agent développeur

dev_agent.rs installe Aider, valide les paramètres, lance Aider dans le projet cible et construit un diff avant/après.

Flux principaux

Flux Entrée frontend Commandes / modules backend Sortie utilisateur
Chat local ChatPage + ChatContext search_web optionnel, puis send_ollama_chat_message et ollama::send_chat_message Réponse assistant affichée dans le chat, avec sources web si SearXNG a été utilisé.
Téléchargement modèle Ollama ChatModelsSection + useChatModelActions pull_ollama_model, ollama pull Modèle visible dans la liste du chat après rafraîchissement.
Génération image ImageGeneratorPage + workflow JSON queue_comfyui_image_generation, API /prompt et /history Image encodée en data URL puis possibilité d'enregistrement.
Modèles ComfyUI ImageModelsSection download_comfyui_model_files, delete_comfyui_model_files Fichiers présents ou supprimés dans ComfyUI/models.
Agent développeur DeveloperAgentPage + paramètres persistés send_developer_agent_message, aider-chat, snapshot de projet Résumé, stdout/stderr tronqués, statut Git et diff détecté.
Installation InstallationSection + useRuntimeOperations install_all, install_ollama, ensure_comfyui_installed, ensure_developer_agent_installed Composants locaux prêts ou message d'erreur dans les logs.

Backend Rust / Tauri

Fichier Responsabilité Points clés
src-tauri/src/main.rs Enregistre les commandes Tauri et arrête les services à la fermeture. start_runtime, start_comfyui, start_searxng, send_ollama_chat_message, queue_comfyui_image_generation.
src-tauri/src/common.rs Types partagés, settings JSON, événements Tauri, streaming stdout/stderr et gestion des processus. emit_runtime_log, emit_runtime_status, run_streaming_command, kill_process_tree.
src-tauri/src/ollama.rs Détection Ollama, démarrage ollama serve, liste/pull/delete de modèles et appel HTTP /api/chat. Gère les timeouts, le décodage HTTP chunked et les chemins configurés.
src-tauri/src/comfyui.rs Détection/démarrage ComfyUI, fichiers modèles, requêtes HTTP ComfyUI, lecture et sauvegarde d'images. Protège les chemins relatifs, attend /history jusqu'au retour d'une image.
src-tauri/src/dev_agent.rs Paramètres Aider, environnement local, exécution de l'agent et diff avant/après. Snapshot limité à 5000 fichiers, 4 Mo au total, 512 Ko par fichier.
src-tauri/src/web_search.rs Paramètres de recherche web et interrogation d'une instance SearXNG compatible JSON. Construit /search?q=...&format=json, limite les résultats transmis au modèle et supporte HTTP/HTTPS via curl ou PowerShell.
src-tauri/src/searxng.rs Installation locale, configuration, démarrage et arrêt de SearXNG. Télécharge l'archive officielle, crée un venv Python 3.11, active le format JSON et lance python -m searx.webapp sur 127.0.0.1:8888.
src-tauri/src/installation.rs Installation Ollama, Open WebUI compatibilité, ComfyUI, Aider, SearXNG, uv et Python. ComfyUI utilise Python 3.13 ; Open WebUI, Aider et SearXNG utilisent Python 3.11.
src-tauri/src/python.rs Détection Python 3.11 et uv, construction des commandes d'installation uv. Supporte macOS, Linux et Windows.
src-tauri/src/openwebui.rs Module de compatibilité Open WebUI : chemins, installation gérée, démarrage et arrêt. Encore présent côté backend, non central dans le parcours UI actuel.
src-tauri/src/system.rs Utilitaire système minimal. Retourne la mémoire totale en Mo.

État, événements et stockage

État frontend

L'état durable de l'interface est réparti entre les contexts de chat et le store launcherStore. Les hooks gardent les états transitoires d'action.

Événements backend

Les processus longs émettent launcher-log pour les logs et launcher-status pour le message de progression et le pourcentage.

Configuration persistée

common.rs lit et écrit launcher-settings.json dans le dossier applicatif Tauri. Il contient les chemins et paramètres de l'agent.

Données locales

ComfyUI, Open WebUI compatibilité, Aider et SearXNG sont installés sous app_data_dir dans des dossiers gérés par Assistia.

Maintenance

Commandes de développement

npm install
npm run dev
npm run tauri dev
npm run build

Points d'attention

  • Les noms de commandes dans src/services/tauris.ts doivent rester alignés avec tauri::generate_handler! dans main.rs.
  • Les messages backend traduits côté frontend passent par src/utils/runtimeMessages.ts. Ajouter une nouvelle chaîne runtime peut nécessiter une clé i18n.
  • Les workflows ComfyUI doivent être référencés dans src/config/imageGeneratorModels.json et disponibles dans src/config/workflows/.
  • La recherche web peut utiliser le SearXNG local installé par Assistia sans Docker. Le fichier settings.yml généré active json dans search.formats.
  • Les téléchargements de modèles ComfyUI doivent utiliser des chemins relatifs sûrs, car comfyui.rs rejette les chemins absolus et les remontées de dossier.
  • L'agent développeur modifie un dossier utilisateur. Le diff affiché repose sur un snapshot texte limité et sur Git lorsqu'il est disponible.
  • Open WebUI est encore installable techniquement, mais toute réintroduction dans l'interface doit reconnecter les pages React, hooks et libellés correspondants.

Cette page est volontairement centrée sur les modules actifs de l'application. Les détails bas niveau restent dans le code source, qui fait autorité en cas d'écart.
